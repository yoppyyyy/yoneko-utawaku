# 🎤 VTuber歌枠ファンサイト — 要件定義書

## 1. プロジェクト概要  

特定VTuberの歌枠配信に特化した非公式ファンサイト。
過去の歌枠配信のセットリストをデータベース化し、楽曲検索やワンクリック再生を提供する。

---

## 2. ゴール

- ファンが「あの曲いつ歌ってたっけ？」をすぐ見つけられる
- 新規ファンが「どんな曲を歌ってるんだろう」を一覧で把握できる
- サムネイル＋ワンクリックでアーカイブの該当箇所を即再生

---

## 3. 技術構成

| 項目 | 選定 |
|------|------|
| フロント | 静的サイト（HTML / CSS / JS） |
| データソース | Google スプレッドシート → ビルド時にJSON生成 |
| データ取得スクリプト | Node.js（Google Sheets API） |
| CI/CD | GitHub Actions（定期 or 手動トリガー） |
| ホスティング | GitHub Pages（無料） |
| デザイン | 推しのイメージカラーベース、レスポンシブ対応 |

### データフロー

```
Google スプレッドシート（メンテナーが編集）
  ↓
GitHub Actions（cron: 毎時 or 手動トリガー）
  ↓ Node.js スクリプトが Sheets API でデータ取得
  ↓ JSON ファイルを生成（streams.json / setlists.json）
  ↓ site/ + data/ をまとめて gh-pages ブランチにデプロイ
  ↓
GitHub Pages で配信
  ↓
閲覧者のブラウザ: fetch("./data/streams.json") → 描画
```

### ブランチ戦略

| ブランチ | 内容 |
|---------|------|
| main | ソースコードのみ（JSON なし）。開発者が編集する |
| gh-pages | ビルド済みファイル一式（HTML/CSS/JS + JSON）。Actions が自動生成・上書き |

- JSONファイルは main ブランチにコミットされない（.gitignore）
- gh-pages ブランチは自動デプロイ用なのでコミット履歴が増えても問題なし
- データの確認は公開URL（`https://{user}.github.io/{repo}/data/streams.json`）または gh-pages ブランチ上で可能

### リポジトリ構成

```
repo/
├── .github/
│   └── workflows/
│       └── deploy.yml            ← GitHub Actions 定義
│                                    （cron or 手動で実行）
│
├── scripts/
│   └── fetch-sheets.js           ← Node.js スクリプト
│                                    （Sheets API → JSON 生成）
│
├── site/                         ← サイトのソースコード
│   ├── index.html
│   ├── css/
│   │   └── style.css
│   └── js/
│       └── app.js                ← fetch("./data/*.json") で読み込み
│
├── .gitignore                    ← data/ を除外
├── package.json
└── README.md
```

※ ビルド時に Actions が `site/` + 生成した `data/` をマージして gh-pages にデプロイ

## 4. MVP機能一覧

### 4-1. 楽曲検索
- 曲名・アーティスト名でインクリメンタルサーチ（リアルタイム絞り込み）
- 検索結果から該当する配信＋タイムスタンプへリンク
- 同じ曲を複数回歌っている場合、全ての配信を表示

### 4-2. 歌枠一覧ページ
- 配信日（新しい順）でカード表示
- 各カードに：サムネイル画像 / 配信日 / タイトル
- カードをクリック → セットリスト詳細を展開 or 遷移

### 4-3. 楽曲一覧ページ
- 全楽曲をリスト表示（曲名 / アーティスト）
- 歌唱回数を自動カウント表示
- ソート：曲名順 / 歌唱回数順 / 最新歌唱日順

### 4-4. ワンクリック再生
- 各曲の横に再生ボタン
- クリックで YouTube を `?t=秒数` 付きで開く
- または、サイト内埋め込みプレーヤーで該当箇所から再生

### 4-5. サムネイル表示
- YouTube動画IDから自動取得（`https://img.youtube.com/vi/{VIDEO_ID}/mqdefault.jpg`）
- 歌枠一覧・検索結果で表示

## 5. データ構造（スプレッドシート）

### シート1: 配信（streams）

| 列 | 内容 | 例 |
|----|------|-----|
| stream_id | 配信の一意ID | S001 |
| date | 配信日 | 2026-04-10 |
| title | 配信タイトル | 【歌枠】12時間耐久！ |
| youtube_url | YouTube URL | https://www.youtube.com/watch?v=xxxxx |
| description | 概要（任意） | 春のアニソン祭り！ |

### シート2: セットリスト（setlists）

| 列 | 内容 | 例 |
|----|------|-----|
| stream_id | 配信ID（シート1と紐付け） | S001 |
| track_no | 曲順 | 1 |
| song_title | 曲名 | 桜 |
| artist | アーティスト名 | コブクロ |
| timestamp | タイムスタンプ | 00:12:45 |
| seconds | 秒数 | 765 |

## 6. ページ構成

```
/ (トップ)
├── 楽曲検索バー（メインUI）
├── 最新の歌枠カード（直近3件）
│
├── /streams （歌枠一覧）
│   └── 配信カード × N（サムネ＋日付＋タイトル）
│       └── 展開 → セットリスト＋再生ボタン
│
└── /songs （楽曲一覧）
    └── 全曲テーブル（検索・ソート可能）
        └── 曲クリック → 歌唱された配信一覧
```

※ 静的サイトのため、実際はSPA的にJSでページ切り替え or 単一ページ内タブ切り替え

## 7. デザイン方針

### 7-1. コンセプト
ゴシック（吸血鬼）× 可愛さ（猫）を両立するダークUIで、視認性を犠牲にしない設計。

### 7-2. カラーパレット（確定）

キャラクター画像から抽出した4グループ構成。CSS カスタムプロパティで管理する。

#### Primary — ロイヤルパープル（瞳・リボン由来）
| トークン | HEX | 用途 |
|---------|---------|------|
| --primary-900 | #2a1f3d | 背景・ベース |
| --primary-700 | #4f3560 | カード背景 |
| --primary-500 | #7b52a0 | ボタン・見出し |
| --primary-300 | #a87ecc | ホバー・リンク |
| --primary-100 | #d4b8e8 | テキスト強調 |

#### Accent — ムーンライトブルー（水色リボン由来）
| トークン | HEX | 用途 |
|---------|---------|------|
| --accent-700 | #3d4f7a | アクセント背景 |
| --accent-500 | #6a7ab2 | アイコン・バッジ |
| --accent-300 | #95a8d4 | サブリンク |

#### Highlight — ゴシックゴールド（金髪由来）
| トークン | HEX | 用途 |
|---------|---------|------|
| --gold-500 | #d4a853 | 再生ボタン・CTA |
| --gold-300 | #ebd2a8 | 見出しアクセント |
| --gold-100 | #f5e6cc | 本文テキスト |

#### Neutral — シャドウ（紫みニュートラル）
| トークン | HEX | 用途 |
|---------|---------|------|
| --neutral-950 | #0a0a12 | 最背面 |
| --neutral-900 | #14121e | カード背景 |
| --neutral-800 | #1e1a2e | ヘッダー・フッター |
| --neutral-600 | #3a3450 | ボーダー・区切り |
| --neutral-100 | #e8e0f0 | 本文テキスト |

### 7-3. 配色理論の根拠
- **類似色調和**: 紫×青（色相環で隣接）→ ゴシックな統一感
- **補色対比**: 紫×金（対向位置）→ CTA・再生ボタンへの視線誘導
- **紫みニュートラル**: 純粋グレーではなく紫を混ぜブランドとの一体感を確保
- **WCAG AA準拠**: N-950背景 × N-100テキスト = コントラスト比約15:1

### 7-4. レイアウト・UI
- **レスポンシブ**: モバイルファースト、PC でも見やすいレイアウト
- **トーン**: ゴシックの洗練さ＋ファンサイトの温かみ
- **サムネイル**: カード型レイアウトで視覚的に配信を一覧できる
- **フォント**: 日本語が美しく表示されるフォント選定（ゴシック系ディスプレイ＋本文）

## 8. 運用フロー

### 新しい歌枠が配信されたら：
1. スプレッドシートの「配信」シートに1行追加
2. 「セットリスト」シートに曲ごとの行を追加（曲名・タイムスタンプ）
3. GitHub Actions が定期実行（毎時）でJSONを再生成 → 自動デプロイ
4. 急ぎの場合は Actions の手動トリガー（Run workflow）で即反映

### メンテナー追加：
- Google スプレッドシートの共有設定で編集権限を付与するだけ
- サイトのデプロイやコードに触る必要なし
