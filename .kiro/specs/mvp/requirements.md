# Requirements: MVP — 夜猫歌枠あーかいぶ

## Overview

Pastel Live所属・夜猫アヤさんの歌枠配信セットリストをデータベース化し、楽曲検索・ワンクリック再生を提供する非公式ファンサイトのMVP。

## Site Identity

- **サイト名**: 夜猫歌枠あーかいぶ
- **サブタイトル（サイト説明）**:
  - Pastel Live所属　夜猫アヤさんの歌枠を検索できるサイトです。
  - このサイトは非公式のファンサイトです。
- **フッター権利表記**（ヘッダーのサブタイトルと重複する「本サイトは非公式…」行は含めない）:
  - 当サイトにて掲載している動画は夜猫アヤ様、及びPastel Live様が制作したものであり、著作権は権利者に帰属します。
  - This is an unofficial fan site. All stream content belongs to its respective creator.
- **サブタイトル表示スタイル**: 本文フォントで表示し、斜体は使用しない

## Goals

- ファンが「あの曲いつ歌ってたっけ？」をすぐ見つけられる
- 新規ファンが「どんな曲を歌ってるんだろう」を一覧で把握できる
- サムネイル＋ワンクリックでアーカイブの該当箇所を即再生

## Scope

### In Scope

- 楽曲検索（インクリメンタルサーチ）
- 歌枠一覧ページ（カード表示＋セットリスト展開）
- 楽曲一覧ページ（ソート・検索可能なテーブル）
- ワンクリック再生（YouTube `?t=秒数` リンク）
- サムネイル自動表示（YouTube動画IDから取得）
- Google Sheets → JSON データパイプライン（GitHub Actions）
- GitHub Pages デプロイ

### Out of Scope

- 歌唱回数ランキング
- アーティスト別・ジャンル別フィルタ
- 月別統計ダッシュボード
- お気に入り機能（ローカルストレージ）
- SNSシェアボタン
- YouTube API連携での新着自動検出
- コメント欄からのセトリ自動取得

## Dependencies

| 依存先 | 用途 | 種別 |
|--------|------|------|
| Google Sheets API | スプレッドシートからデータ取得 | 外部API |
| YouTube Data API v3 | スプレッドシート元ネタ（歌枠配信一覧）取得 | 外部API |
| YouTube (img.youtube.com) | サムネイル画像取得 | 外部リソース |
| GitHub Actions | CI/CD（JSON生成＋デプロイ） | インフラ |
| GitHub Pages | 静的サイトホスティング | インフラ |

## Functional Requirements

### FR-1: 楽曲検索

- **FR-1.1**: 曲名・アーティスト名でインクリメンタルサーチ（リアルタイム絞り込み）ができる
- **FR-1.2**: 検索結果から該当する配信＋タイムスタンプへのリンクを表示する
- **FR-1.3**: 同じ曲を複数回歌っている場合、全ての配信を表示する

### FR-2: 歌枠一覧

- **FR-2.1**: 配信日（新しい順）でカード形式で一覧表示する
- **FR-2.2**: 各カードにサムネイル画像・配信日・タイトルを表示する
- **FR-2.3**: カードクリックでセットリスト詳細を展開表示する
- **FR-2.4**: ソート切り替え — 新しい順 / 古い順（初期表示は「新しい順」アクティブ）
- **FR-2.5**: 歌枠カードを展開して表示されるセットリストには、曲順番号（`track_no`）や内部ID（`stream_id` 等）を表示しない。各行は「曲名・アーティスト・タイムスタンプ・再生ボタン」のみで構成する

### FR-3: 楽曲一覧

- **FR-3.1**: 全楽曲をリスト表示する（曲名・アーティスト）
- **FR-3.2**: 歌唱回数を自動カウント表示する（Stats Barにはユニーク曲数カウンタを表示しない）
- **FR-3.3**: ソート切り替え — 最新順 / 歌唱回数順 / 曲名順（この順で左から配置、初期表示は「最新順」アクティブ）

### FR-4: ワンクリック再生

- **FR-4.1**: 各曲の横に再生ボタンを表示する
- **FR-4.2**: クリックで YouTube を `?t=秒数` 付きで新タブに開く

### FR-5: サムネイル表示

- **FR-5.1**: YouTube動画IDから `https://img.youtube.com/vi/{VIDEO_ID}/mqdefault.jpg` でサムネイルを自動取得する
- **FR-5.2**: 歌枠一覧・検索結果でサムネイルを表示する

### FR-6: データパイプライン

- **FR-6.1**: Node.jsスクリプトでGoogle Sheets APIからデータを取得する
- **FR-6.2**: `streams.json` と `setlists.json` の2つのJSONファイルを生成する
- **FR-6.3**: GitHub Actionsで定期実行（毎時）または手動トリガーで実行する
- **FR-6.4**: 生成したJSONとサイトファイルを gh-pages ブランチにデプロイする

### FR-6b: スプレッドシート元データ取得スクリプト

メンテナーが歌枠配信一覧をスプレッドシートに流し込む前段として、YouTubeチャンネルから配信メタデータを半自動取得するためのローカル実行スクリプトを提供する。

- **FR-6b.1**: Python スクリプト `scripts/fetch-youtube/fetch_utawaku.py` をローカル実行して、対象チャンネル（夜猫アヤ）のアップロード動画一覧を YouTube Data API v3 経由で取得する
- **FR-6b.2**: タイトルにキーワード「歌枠」を含む動画のみを抽出する
- **FR-6b.3**: 各配信について「配信日（YYYY-MM-DD HH:MM）・タイトル・URL・概要（先頭500文字、改行はスペース置換）」を CSV (`utawaku_list.csv`, UTF-8 BOM付き) に出力する
- **FR-6b.4**: 出力は配信日の昇順でソートする
- **FR-6b.5**: API キー・チャンネルID・検索キーワード・出力ファイル名はスクリプト先頭の定数で設定する
- **FR-6b.6**: 本スクリプトは運用補助ツールであり、サイトのビルドパイプライン（FR-6）には組み込まない。出力 CSV はメンテナーが手動でスプレッドシートに取り込む
- **FR-6b.7**: API キーは公開リポジトリにコミットしない（`.gitignore` もしくは環境変数化などで保護する）

### FR-8: 今日の3曲タブ

- **FR-8.1**: ナビタブに「今日の3曲」タブを追加する
- **FR-8.2**: 全セットリストから重複なしで3曲をランダムに選出する
- **FR-8.3**: 各カードには、その曲が歌われた配信（複数回歌われている場合は1回分をランダム選択）の **YouTubeサムネイル** を表示する
- **FR-8.4**: 各カードに曲名とアーティスト名を表示する。文字色は歌枠カードの配信タイトルと同じ金色（`--gold-100`）とする
- **FR-8.5**: 各カードに再生ボタンを表示し、クリックするとカード本体の別操作なしで対象配信の該当タイムスタンプ（`?t=秒数`）が新タブで開く
- **FR-8.6**: ランダム抽選はブラウザの初回ロード時のみ行い、タブ切替では再抽選しない（ページをリロードすると新しい3曲になる）
- **FR-8.7**: 「今日の3曲」タブがアクティブな間、検索入力欄（`.search-section`）そのものを非表示にする。他タブへ切り替えると再び表示される。ナビタブ（`.nav-tabs`）は引き続き表示する

- **FR-7.1**: SPA的にJSでページ切り替え（トップ / 歌枠一覧 / 楽曲一覧）
- **FR-7.2**: トップページに検索バーと最新の歌枠カード（直近3件）を表示する

## Non-Functional Requirements

### NFR-1: パフォーマンス

- **NFR-1.1**: 静的サイトとして配信し、サーバーサイド処理を不要とする
- **NFR-1.2**: JSONデータを `fetch()` でクライアント側で読み込み描画する

### NFR-1b: 閲覧体験（スクロール挙動）

- **NFR-1b.1**: ページスクロール時、検索バーとナビタブはビューポート上端に固定（sticky）され、その下のコンテンツ（歌枠一覧・楽曲一覧）のみがスクロールする。ヘッダー／Stats Bar は通常どおりスクロールアウトする
- **NFR-1b.2**: stickyになったブロックは背景が透けないよう body 同色で不透明化し、下のカードと視覚的に重ならないようにする（下端は1pxボーダーのみで境界を示し、ドロップシャドウは使わない）

### NFR-2: デザイン

- **NFR-2.1**: 落ち着いたダークトーン（深紺ベース）にセージグリーンの挿し色で構成するダークUI
- **NFR-2.2**: 確定済み5色パレットを使用する
  - No1 `#262D40` 深紺 — 背景
  - No2 `#97AE73` セージ — 強調色（タイトル、再生ボタン、アクティブ指標）
  - No3 `#3F8B92` ティール — セカンダリアクセント（数値、リンク、副次強調）
  - No4 `#8996AC` 青灰 — ボーダー／ミュートテキスト
  - No5 `#8AAEA9` 淡緑灰 — 本文テキスト
  - 「面」（カード／タブ／テーブル行／アクティブ状態）は上記5色のrgba半透明オーバーレイで階層化する（新たな色は追加しない）
  - 背景装飾オーバーレイ（旧パターン／ラジアルグラデ）は使用しない
- **NFR-2.3**: モバイルファーストのレスポンシブ対応
- **NFR-2.4**: WCAG AA準拠のコントラスト比を確保する
- **NFR-2.5**: Stats Bar のラベル（STREAMS / SONGS）は背景から十分に視認できる明度で表示する（ミュート系の暗色は使わない）
- **NFR-2.6**: フッターの権利表記テキスト（2行とも）は白系（本文色）で表示し、過度に沈み込ませない
- **NFR-2.7**: 歌枠一覧グリッドはデスクトップ幅で最大4カード/行までとし、サムネイルは従来より一段コンパクトに収める

### NFR-3: 運用性

- **NFR-3.1**: データ更新はスプレッドシート編集のみで完結する（コード変更不要）
- **NFR-3.2**: メンテナー追加はスプレッドシートの共有設定のみで可能

## Data Model

### streams（配信）

| フィールド | 型 | 説明 | 例 |
|-----------|-----|------|-----|
| stream_id | string | 配信の一意ID | S001 |
| date | string (YYYY-MM-DD) | 配信日 | 2026-04-10 |
| title | string | 配信タイトル | 【歌枠】12時間耐久！ |
| youtube_url | string | YouTube URL | https://www.youtube.com/watch?v=xxxxx |
| description | string? | 概要（任意） | 春のアニソン祭り！ |

### setlists（セットリスト）

| フィールド | 型 | 説明 | 例 |
|-----------|-----|------|-----|
| stream_id | string | 配信ID（streamsと紐付け） | S001 |
| track_no | number | 曲順 | 1 |
| song_title | string | 曲名 | 桜 |
| artist | string | アーティスト名 | コブクロ |
| timestamp | string (HH:MM:SS) | タイムスタンプ | 00:12:45 |
| seconds | number | 秒数 | 765 |
